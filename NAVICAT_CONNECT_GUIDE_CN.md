# Navicat 连接远程数据库教程 (SSH 隧道方式)

为了安全起见，我们**不建议**直接把 MySQL 的 3306 端口暴露给公网。
最安全、最推荐的方法是使用 Navicat 自带的 **SSH 隧道** 功能。它的原理是先通过 SSH 登录服务器，再在服务器内部连接数据库。

## 1. 新建连接
打开 Navicat，点击左上角的 **连接 (Connection)** -> **MySQL**。

## 2. 设置 SSH 选项卡 (关键步骤！)
先点击 **SSH** 选项卡（不要先填常规）：
1.  **使用 SSH 通道 (Use SSH Tunnel)**：勾选它。
2.  **主机 (Host)**：填你的服务器 IP `159.75.135.120`。
3.  **端口 (Port)**：保持 `22`。
4.  **用户名 (Username)**：填 `ubuntu`。
5.  **验证方法 (Authentication Method)**：选择 `密码` (Password)。
6.  **密码 (Password)**：填你的服务器登录密码。
7.  *点击下方的“测试连接”按钮，如果提示成功，说明 SSH 通了。*

## 3. 设置常规选项卡
回到 **常规 (General)** 选项卡：
1.  **连接名 (Connection Name)**：随便填，比如 `FoodAI-Remote`。
2.  **主机 (Host)**：这里要填 `localhost` 或 `127.0.0.1`。
    *   *解释：因为走了 SSH 隧道，对于服务器来说，我们就是“在本地”连接。*
3.  **端口 (Port)**：`3306`。
4.  **用户名 (Username)**：`food_user`。
5.  **密码 (Password)**：`Food_2024@Pass` (或者你自己改过的数据库密码)。

## 4. 连接成功
点击下方的 **测试连接**，应该就能看到“连接成功”了！
确定后双击左侧的连接名，就能看到 `food_db` 数据库了。
