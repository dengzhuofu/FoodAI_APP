我将按照新的 TODO List 顺序，分模块推进前端与后端的对接开发。

## 阶段一：个人中心与库存模块 (Profile & Inventory)
首先打通用户基础数据和核心工具功能，这是用户使用频率较高的部分。

1.  **API 封装**:
    *   创建 `src/api/profile.ts`: 封装获取和更新用户资料、风味画像的接口。
    *   创建 `src/api/inventory.ts`: 封装冰箱食材和购物清单的增删改查接口。
2.  **页面对接**:
    *   `ProfilePage.tsx`: 读取用户信息展示，处理退出登录。
    *   `FlavorProfilePage.tsx`: 读取和保存用户的口味偏好、过敏源等。
    *   `MyKitchenPage.tsx`: 替换 Mock 数据，改为从后端加载冰箱库存，实现真实的添加和删除。
    *   `ShoppingListPage.tsx`: 对接购物清单的同步逻辑。

## 阶段二：内容与社交模块 (Content & Social)
接着完善内容消费和互动体验。

1.  **API 封装**:
    *   创建 `src/api/content.ts`: 封装菜谱/餐厅的获取、详情、评论、收藏接口，以及首页推荐流接口。
2.  **页面对接**:
    *   `RecommendScreen.tsx`: 接入 `/api/v1/feed/recommend` 接口，展示真实的推荐列表。
    *   `RecipeDetailPage.tsx` & `RestaurantDetailPage.tsx`: 根据 ID 加载详情数据，实现真实的收藏和评论功能。

## 阶段三：AI 服务模块 (AI Integration)
最后对接应用的核心差异化功能。

1.  **API 封装**:
    *   创建 `src/api/ai.ts`: 封装所有 AI 相关的 POST 请求 (文本生菜谱、图生菜谱等)。
2.  **页面对接**:
    *   `TextToRecipeFeature.tsx`: 实现用户输入描述 -> 调用 API -> 展示生成结果。
    *   `FridgeToRecipeFeature.tsx`: 获取冰箱库存 -> 调用 API -> 展示推荐菜谱。
    *   其他 AI 功能页面的对接。
